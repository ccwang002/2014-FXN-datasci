Data ETL summary and notes

  • Split your ETL into separate stages
  • Use a familiar programming language, which should be trivial to handle following scenarios
    • Data I/O: CSV/TSV/JSON file parsing; database connection
    • Daata cleaning: Unicode conversion; extraction from certain fields
    • Parsing: time, location data
    • Web scraping (if needed)
  • No clue? Use R/Python/Scala(for MapReduce)

Some terms need to know

  • Supervised learning: classification
  • Unsupervised learning: clustering
  • Model selection and evaluation
  • Dimension reduction

Before we start to speak statistics

Statistics is about

  • Not merely mean and variance (also known as descriptive statistics)
  • Sampling and finding representative samples (otherwise, biased)
  • Modeling for the specific data structure
  • Avoid over fitting or complicated models (model simplification)
  • Insights about data's mean and variance (or domain knowledge)

Thoughts about the previous model

  • How to test our model? train/test, cross-validation
  • What happen if more data come in?
    • Retrain the model? Feasible if is the case 100M + 100/hr?
    • Can my model handle exception? Ex. A classifier of oranges and apples, but here comes a bananas
  • Is my model explainable?
    Ex. neural network, deep learning, PCA dimension reduction
